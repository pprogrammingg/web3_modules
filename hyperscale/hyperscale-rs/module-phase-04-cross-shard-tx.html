Proof snippets below are trimmed from the hyperscale-rs tree. Same ingress path as Phase 1 — multi-shard is “more shards in the loop.” Ellipsis (...) hides boilerplate; follow data-file links for full sources.

In-repo proof — declared reads ∪ writes → per-shard gossip batch

// crates/node/src/io_loop/step/tx_validation.rs — handle_submit_transaction
let shards: BTreeSet<ShardGroupId> = tx.declared_reads().iter()
    .chain(tx.declared_writes().iter())
    .map(|node_id| shard_for_node(node_id, num_shards)).collect();
for shard in shards {
    self.enqueue_tx_for_gossip(shard, Arc::clone(&tx));
}
// crates/types/src/topology/snapshot.rs
pub fn is_cross_shard_transaction(&self, tx: &RoutableTransaction) -> bool {
    self.all_shards_for_transaction(tx).len() > 1
}
