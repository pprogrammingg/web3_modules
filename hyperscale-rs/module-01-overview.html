1. What is Blockchain?

Core Concept

A blockchain is a distributed ledger that maintains a continuously growing list of records (blocks) that are linked and secured using cryptography. Each block contains:

  • Transactions - The actual data being recorded
  • Hash - A cryptographic fingerprint of the block
  • Previous block hash - Links to the previous block, forming a chain
Key Insight: The "chain" in blockchain comes from each block referencing the previous block's hash. If someone tries to modify an old block, all subsequent blocks become invalid, making tampering extremely difficult.

Why Blockchain Matters

Blockchains provide:

  • Decentralization - No single point of control or failure
  • Immutability - Once recorded, data is extremely difficult to change
  • Transparency - All participants can verify the ledger
  • Trust - Cryptographic proofs ensure data integrity

Types of Blockchains

  • Public - Anyone can join and participate (Bitcoin, Ethereum)
  • Permissioned - Only approved participants can join
  • Consortium - Controlled by a group of organizations

2. What is Hyperscale-rs?

Definition

Hyperscale-rs is a Rust implementation of the Hyperscale consensus protocol - a Byzantine Fault Tolerant (BFT) consensus system designed for high-throughput, sharded blockchain networks.

Key Point: Hyperscale-rs is NOT a full blockchain platform. It's a consensus layer - the "engine" that decides which transactions get included and in what order.

What Hyperscale-rs IS:

  • ‚úÖ A BFT consensus protocol implementation
  • ‚úÖ Designed for high throughput and low latency
  • ‚úÖ Supports sharding (dividing the network into parallel chains)
  • ‚úÖ Handles cross-shard transactions securely
  • ‚úÖ Pure consensus logic (no I/O, no locks, no async in core)
  • ‚úÖ Deterministic and testable

What Hyperscale-rs is NOT:

  • ‚ùå A full blockchain platform (like Ethereum or Bitcoin)
  • ‚ùå A cryptocurrency (no native token)
  • ‚ùå A smart contract platform (though it integrates with Radix Engine)
  • ‚ùå A wallet or user-facing application
  • ‚ùå Production-ready (still work in progress)

Design Philosophy

Hyperscale-rs follows several key design principles:

  1. Pure Consensus Layer - No I/O, no locks, no async in consensus logic
  2. State Machine Model - Everything is a deterministic state machine
  3. Event-Driven - Events flow in, actions flow out
  4. Deterministic Testing - Same code runs in simulation and production

4. Setting Up Your Environment

Prerequisites

  1. Install Rust (if not already installed):
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
  2. Install Build Dependencies:
    • macOS: brew install llvm protobuf openssl pkg-config
    • Linux: sudo apt-get install clang lld pkg-config protobuf-compiler libssl-dev
  3. Clone the Repository:
    git clone --recurse-submodules https://github.com/flightofthefox/hyperscale-rs.git cd hyperscale-rs
  4. Build the Project:
    cargo build --release
  5. Run Tests:
    cargo test
Important: Make sure to clone with --recurse-submodules as the project uses git submodules for Radix Engine dependencies.
